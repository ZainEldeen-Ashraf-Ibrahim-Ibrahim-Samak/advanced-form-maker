# Data Model: Native Submission Screen

## Entity: SubmissionSession
Represents the active token-based context used by the mobile native submission flow.

Fields:
- token: string (sensitive, secure storage only)
- submissionId: string | null
- formTemplateId: string
- mode: enum(new, draft, needs_rewrite)
- formVersion: string
- submissionUpdatedAt: string | null
- locale: enum(en, ar)
- canSubmit: boolean (derived)
- isOnline: boolean (derived from connectivity)

Validation rules:
- token must be present before any API call.
- mode is derived from GET /api/submissions/{token} response.
- canSubmit = all required validations pass AND required uploads complete AND connectivity true.

State transitions:
- initializing -> loading_context
- loading_context -> ready_editable
- ready_editable -> blocked_validation | blocked_offline | blocked_required_media | submitting
- submitting -> success | stale_conflict | recoverable_error | terminal_error
- any state -> invalid_or_expired_token (clears secure local session)

## Entity: LocalDraft
Encrypted persisted draft used for interruption recovery and offline editing continuity.

Fields:
- draftId: string
- tokenRef: string (indirect reference; no plain token logging)
- contacts: ContactRecord[]
- fieldResponses: FieldResponse[]
- mediaQueue: MediaUploadItem[]
- lastSyncedFormVersion: string
- lastSavedAt: string
- schemaVersion: number

Validation rules:
- Persist only encrypted payload for sensitive content.
- Clear on successful submit/resubmit.
- Clear on invalid/expired/ineligible token response.

Relationships:
- One SubmissionSession has zero or one LocalDraft.
- One LocalDraft contains many ContactRecord and many FieldResponse records.

## Entity: ContactRecord
User contact unit required by submission flow.

Fields:
- id: string
- name: string
- email: string | null
- phone: string | null
- contact: string | null
- role: string | null
- notes: string | null
- mediaUrl: string | null
- mediaPublicId: string | null

Validation rules:
- At least one contact record is required.
- email must match EMAIL_REGEX when provided.
- phone must match PHONE_REGEX when provided.
- name must match NAME_REGEX when provided.
- contact text/address must match TEXT_REGEX when provided.

## Entity: FieldResponse
Submission response value mapped to a form field definition.

Fields:
- fieldDefinitionId: string
- value: string | number | string[] | null
- mediaUrl: string | null
- mediaPublicId: string | null
- mediaItems: { url: string; publicId: string }[]

Validation rules:
- Required fields must include either text value, mediaUrl, or mediaItems.
- Dropdown multi-select fields must send array values only and options must exist in allowed options set.
- Regex-constrained fields must satisfy assigned regex type.

Relationships:
- Many FieldResponse items belong to one SubmissionSession.
- Each FieldResponse references one backend field definition snapshot or live definition.

## Entity: MediaUploadItem
Tracks local and remote upload status for required/optional attachments.

Fields:
- fieldDefinitionId: string
- localUri: string
- status: enum(pending, uploading, uploaded, failed)
- uploadedUrl: string | null
- uploadedPublicId: string | null
- required: boolean
- lastError: string | null

Validation rules:
- Final submit/resubmit is blocked while any required item is pending/uploading/failed.
- Only uploadedUrl/uploadedPublicId references are sent in submission payload.

## Entity: SubmissionOutcome
Result of submit/resubmit attempt.

Fields:
- kind: enum(success, validation_error, stale_conflict, unauthorized, server_error)
- code: string | null
- message: string
- retryable: boolean

Validation rules:
- stale_conflict must show explicit refresh/retry guidance.
- unauthorized or invalid token outcomes must trigger secure local cleanup.

## Relationship Summary
- SubmissionSession 1..1 -> 0..1 LocalDraft
- SubmissionSession 1..* -> ContactRecord
- SubmissionSession 1..* -> FieldResponse
- FieldResponse 0..* -> MediaUploadItem (for media-capable fields)
- SubmissionOutcome generated by submit/resubmit transition from SubmissionSession
