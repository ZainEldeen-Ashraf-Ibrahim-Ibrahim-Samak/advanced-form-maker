## Design Context

### Users
Form administrators — managing submissions, reviewing data, building form templates — in a bilingual (Arabic/English, RTL/LTR) professional context. Extended desktop sessions; needs to feel trustworthy, not playful.

### Brand Personality
Precise · Credible · Arabic-rooted. The interface should feel like a well-made institutional tool — not a startup, not a consumer app. Jewel tones convey richness without being decorative.

### Aesthetic Direction
- **Theme**: Light mode default (dark mode supported via toggle)
- **Primary color**: Deep jewel emerald-teal (oklch hue ~168) — professional, Arabic-appropriate, not the SaaS blue/purple cliché
- **Accent**: Warm amber/gold (hue ~75) — classical Arabic color pairing with green
- **Neutrals**: All tinted toward the brand hue for subconscious cohesion
- **Sidebar**: Deep jewel panel — distinct from the page surface, creates clear information hierarchy
- **Anti-references**: Generic SaaS gray, cyan-on-dark AI palettes, purple gradient marketing sites

### Theming Allowances
The color system is driven by two CSS primitives on `:root`:
- `--brand-hue` (default 168 = emerald)
- `--brand-chroma` (default 0.105)
- `--warm-hue` (default 75 = amber)

Four alternate themes are available as HTML classes:
- `.theme-amber` — warm amber brand
- `.theme-sapphire` — deep sapphire
- `.theme-wine` — rich burgundy
- `.theme-gold` — antique gold

### Design Principles
1. **Restraint over decoration** — color earns its place; the 60-30-10 rule is enforced
2. **Arabic-first typography** — RTL layouts must feel as native as LTR
3. **Depth through surface, not shadow** — elevation differences use lightness steps, not box-shadow
4. **No AI fingerprints** — no border-left accent stripes, no gradient text, no cyan-on-dark glow
5. **Every state matters** — empty, loading, error states are as intentional as the filled state
